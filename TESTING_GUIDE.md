# Foldercraft Complete Testing Guide (Copy-Paste Ready)

This guide helps you test the full `foldercraft` package feature-by-feature using PowerShell on Windows.

All commands are written so you can copy and paste directly.
This also covers automatic per-directory `README.md` generation.

---

## 0) Test setup (one time)

Open PowerShell and run:

```powershell
cd "C:\PERSONAL PROJECTS\foldercraft"
npm install
npm run build
npm link
foldercraft --help
```

Expected:

- `foldercraft --help` prints CLI usage and options.

---

## 1) Create clean test workspace

```powershell
$TestRoot = "$env:TEMP\foldercraft-e2e-tests"
if (Test-Path $TestRoot) { Remove-Item -Recurse -Force $TestRoot }
New-Item -ItemType Directory -Path $TestRoot | Out-Null
Set-Location $TestRoot
Write-Host "Using test root: $TestRoot"
```

### 3.4 Auto README generation (root + nested folders)

```powershell
foldercraft --framework react --path ".\readme-check"
```

Expected:

- All three `Test-Path` checks return `True`.

## 8) Custom config only mode

Create config file:

```powershell
@'
{
  "src": {
    "components": {},
    "index.js": "file",
    "config.js": "export const app = 'foldercraft';\n"
  },
  "docs": {},
  "README.md": {
    "$file": "# Project\nGenerated by foldercraft.\n"
  }
}
'@ | Set-Content -Encoding UTF8 ".\custom-structure.json"
```

Generate from config:

```powershell
foldercraft --config ".\custom-structure.json" --path ".\config-project"
Get-ChildItem -Recurse ".\config-project" | Select-Object FullName
Get-Content ".\config-project\README.md"
```

Expected:

- Structure matches config exactly.

---

## 9) Framework + config merge (`--extend`)

Create override config:

```powershell
@'
{
  "src": {
    "features": {},
    "app.config.js": "export default { mode: 'test' };\n"
  },
  ".env.example": "file"
}
'@ | Set-Content -Encoding UTF8 ".\extend-overrides.json"
```

Run extend:

```powershell
foldercraft --framework react --config ".\extend-overrides.json" --extend --path ".\extend-react"
Test-Path ".\extend-react\src\features"
Test-Path ".\extend-react\.env.example"
Get-ChildItem -Recurse ".\extend-react\src" | Select-Object FullName
```

Expected:

- Base React template + custom override nodes are present together.

## 10) Existing target behavior (default/merge/overwrite)

### 10.1 Default behavior should fail if target exists

```powershell
foldercraft --framework node --path ".\exists-default"
foldercraft --framework node --path ".\exists-default"
```

Expected:

- Second command fails because target already exists.

### 10.2 Merge behavior

```powershell
foldercraft --framework node --path ".\exists-merge"
"keep-me" | Set-Content ".\exists-merge\manual.txt"
"my custom readme" | Set-Content ".\exists-merge\README.md"
foldercraft --framework node --path ".\exists-merge" --merge --add-dir "src/jobs"
Test-Path ".\exists-merge\manual.txt"
Test-Path ".\exists-merge\src\jobs"
Get-Content ".\exists-merge\README.md"
```

Expected:

- `manual.txt` remains.
- new folder (`src/jobs`) is added.
- existing `README.md` content stays unchanged.

### 10.3 Overwrite behavior

```powershell
foldercraft --framework node --path ".\exists-overwrite"
"delete-me" | Set-Content ".\exists-overwrite\manual.txt"
foldercraft --framework node --path ".\exists-overwrite" --overwrite
Test-Path ".\exists-overwrite\manual.txt"
```

Expected:

- `manual.txt` should be gone after overwrite (`False`).
